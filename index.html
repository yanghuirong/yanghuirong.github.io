configTomcatAtEclipse

唔，今天又配置环境了0 0.好不容易想蹭个页面的小功能做一做，所以需要重新配置一下环境。用intellij配置的话，我不知道那种VM argument写在哪里。因为要连数据库嘛。
以前用Eclipse配置过一次了，所以这次还是Eclipse配置环境吧。大概花了一小时吧。写一下需要做的步骤和遇到的问题。
问题1，在eclipse里面没找到新建web project
在eclipse的右上角有一个Java EE | Java 切换到Java EE就好了。
（我反复打开了很多次- - 好坑 我就说我下的就是Eclipse EE嘛。）
问题2，在我的Tomcat/webapps里面打死也没找到我部署的项目。
首先，需要在跟console位置差不多的地方找到server，新建一个server，把项目导入进去，然后双击新建好的server，选择第二个Use Tomcat Installation
Server Path 就是安装 Tomcat的路径
Deploy Path 我是选的 webapps
问题3，tomcat端口被占用
参考昨天的那个ps -ef|grep指令来弄吧~
问题4，run configuration里面的配置直接复制过来是有问题的
仔细检查一下，不需要反斜杠

config里面的config.xml里面的目标数据库改一下。

VS2010 - VS2010 SP1
dll -> c/windows
MYSQL upgrade

Tomcat

8.5
(题外话：今天好烦躁，出差了，换了一个环境，之前用intellij搭建的超级简单的项目居然出问题了，localhost:8080访问不了之类的，同时127.0.0.1:8080又可以访问)
（后来后来，通过程序猿鼓励师的帮忙，首先解决了tomcat 不能 shutdown的问题，后来又解决了找不到java class的问题，后来又有别的问题。解决方案如下）
出现的错误 tomcat,intellij
问题1，端口被占用，不能使用./shutdown.sh关闭Tomcat
localhost没有和那个127.0.0.1联系一起 去/etc/hosts 里面绑定一下（之前的mysites和127.0.0.1绑定了，已经被我删掉了）
问题2，端口被占用
ps -ef|grep tomcat 看看被占用的端口是什么 比如1066
然后 kill -9 1066
问题3，web.xml找不到Test的问题
就是对src目录进行mark dictionary as source root
问题4，output path有问题
应该要改为 /Users/yanghuirong/Documents/web_workspace/test/web/WEB-INF/classes
（把鼠标放在test的根目录 然后按花❀键和回车 设置path）
